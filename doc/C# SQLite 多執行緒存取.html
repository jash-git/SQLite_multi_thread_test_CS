🧩 一、SQLite 的多執行緒原理

SQLite 是輕量級資料庫，預設是針對單執行緒或單進程優化的。
要讓多執行緒同時操作，需要考慮以下幾層設定：

層級重點說明
SQLite 引擎層是否開啟「多執行緒模式」或「序列化模式」
Connection 層每個執行緒要不要共用同一個連線（Connection）
Transaction 層同步機制：同時讀、同時寫會被鎖定
檔案層SQLite 寫入時是以整個檔案鎖定為單位（非 row lock）

✅ 二、建議設定方式

🔹 1. 使用 WAL 模式（Write-Ahead Logging）

這是多執行緒存取同一 SQLite 檔案最重要的一步。
它允許多執行緒同時讀取資料（只有寫入會鎖）。

csharp
using var connection = new SqliteConnection("Data Source=mydb.db;");
connection.Open();

using var command = connection.CreateCommand();
command.CommandText = "PRAGMA journal_mode=WAL;";
command.ExecuteNonQuery();

✅ WAL 模式可同時多讀單寫，是多執行緒場景的首選。

🔹 2. 每個執行緒使用「自己的連線」

不要共用同一個 SqliteConnection。
因為同一個連線物件不是 thread-safe。

csharp
Parallel.For(0, 10, i =>
{
    using var conn = new SqliteConnection("Data Source=mydb.db;");
    conn.Open();

    using var cmd = conn.CreateCommand();
    cmd.CommandText = $"INSERT INTO log_table (msg) VALUES ('Thread {i}')";
    cmd.ExecuteNonQuery();
});

🔹 3. 設定連線字串中的同步與快取策略

csharp
var connStr = new SqliteConnectionStringBuilder
{
    DataSource = "mydb.db",
    Mode = SqliteOpenMode.ReadWriteCreate,
    Cache = SqliteCacheMode.Shared
}.ToString();

Cache=Shared 可以讓同一進程內多個連線共享頁面快取，減少鎖定衝突。

🔹 4. 如果仍發生鎖定，可以加上重試機制

SQLite 在寫入衝突時可能丟出 SQLiteBusyException。
你可以加入簡單的重試：

csharp
for (int retry = 0; retry < 3; retry++)
{
    try
    {
        using var conn = new SqliteConnection(connStr);
        conn.Open();
        using var cmd = conn.CreateCommand();
        cmd.CommandText = "INSERT INTO log_table (msg) VALUES ('test')";
        cmd.ExecuteNonQuery();
        break; // success
    }
    catch (SqliteException ex) when (ex.SqliteErrorCode == 5) // database is locked
    {
        Thread.Sleep(100 * (retry + 1));
    }
}

⚙️ 三、SQLite 執行緒模式（僅供理解）

可在開啟資料庫時查詢：

sql
PRAGMA threads;

SQLite 有三種模式（通常不用手動切換）：

模式說明
single-thread不可多執行緒
multi-thread同一 connection 不可共用，不同 connection 可同時操作
serialized完全 thread-safe（但效能略低）

C# 的 Microsoft.Data.Sqlite 預設是使用 multi-thread，已足夠安全。

🧠 四、整合建議

如果你要多執行緒存取 SQLite，同時確保穩定性與效能：

✅ WAL 模式
✅ 每個執行緒各自建立連線
✅ 使用 Cache=Shared
✅ 適度加上重試機制
✅ 避免長時間 Transaction 鎖定整張表

💻 完整範例程式（.NET 7+ 可執行）

csharp
using System;
using System.IO;
using System.Threading;
using System.Threading.Tasks;
using Microsoft.Data.Sqlite;

class Program
{
    static void Main()
    {
        const string dbFile = "multi_thread_test.db";

        // 若檔案不存在就初始化資料庫
        if (!File.Exists(dbFile))
        {
            InitializeDatabase(dbFile);
        }

        // 啟用 WAL 模式（支援多執行緒多讀單寫）
        using (var conn = new SqliteConnection($"Data Source={dbFile}"))
        {
            conn.Open();
            using var cmd = conn.CreateCommand();
            cmd.CommandText = "PRAGMA journal_mode=WAL;";
            var mode = cmd.ExecuteScalar();
            Console.WriteLine($"WAL 模式啟用結果: {mode}");
        }

        Console.WriteLine("開始平行寫入...");

        var connStr = new SqliteConnectionStringBuilder
        {
            DataSource = dbFile,
            Cache = SqliteCacheMode.Shared,
            Mode = SqliteOpenMode.ReadWrite
        }.ToString();

        // 使用 20 個執行緒平行寫入資料
        Parallel.For(0, 20, i =>
        {
            for (int j = 0; j < 5; j++)
            {
                WriteWithRetry(connStr, i, j);
            }
        });

        Console.WriteLine("寫入完成。");
    }

    static void InitializeDatabase(string dbFile)
    {
        using var conn = new SqliteConnection($"Data Source={dbFile}");
        conn.Open();
        using var cmd = conn.CreateCommand();
        cmd.CommandText = @"
            CREATE TABLE log_table (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                thread_id INTEGER,
                msg TEXT,
                created_at TEXT
            );
        ";
        cmd.ExecuteNonQuery();
        conn.Close();

        Console.WriteLine("資料庫已初始化。");
    }

    static void WriteWithRetry(string connStr, int threadId, int seq)
    {
        const int maxRetry = 5;
        for (int retry = 0; retry < maxRetry; retry++)
        {
            try
            {
                using var conn = new SqliteConnection(connStr);
                conn.Open();
                using var cmd = conn.CreateCommand();

                cmd.CommandText = "INSERT INTO log_table (thread_id, msg, created_at) VALUES (@t, @m, datetime('now'))";
                cmd.Parameters.AddWithValue("@t", threadId);
                cmd.Parameters.AddWithValue("@m", $"Thread {threadId} - record {seq}");
                cmd.ExecuteNonQuery();

                return; // 成功就跳出
            }
            catch (SqliteException ex) when (ex.SqliteErrorCode == 5) // database is locked
            {
                int delay = 100 * (retry + 1);
                Console.WriteLine($"[Thread {threadId}] 資料庫被鎖定，重試第 {retry + 1} 次 (延遲 {delay}ms)");
                Thread.Sleep(delay);
            }
            catch (Exception ex)
            {
                Console.WriteLine($"[Thread {threadId}] 錯誤: {ex.Message}");
                return;
            }
        }
    }
}

ChatGPT 說：

這個錯誤是因為 SQLitePCL 的初始化沒有正確設定。這通常會出現在使用 Microsoft.Data.Sqlite 或其他 SQLite 包的時候，特別是當 SQLite 的底層提供者 (SQLitePCL.raw) 還沒被初始化的情況。

為了解決這個問題，你需要在程式啟動時明確初始化 SQLitePCL.Batteries，這樣才能使用 SQLite。

🛠️ 解決方法

在你的 Main 方法或程式的初始化部分，加入以下程式碼來初始化 SQLitePCL.Batteries：

csharp
using SQLitePCL;

class Program
{
    static void Main()
    {
        // 初始化 SQLitePCL 底層提供者
        Batteries.Init();

        const string dbFile = "multi_thread_test.db";
        // 剩下的程式邏輯...
    }
}

💡 為何需要這個初始化？

SQLitePCL.raw 是 SQLite 的低階提供者，SQLitePCL.Batteries.Init() 會自動幫你設置並載入必要的 SQLite 驅動程式。若沒這麼做，會出現你現在看到的錯誤訊息。
